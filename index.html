Technical Design

Clean layered architecture

Decoupled frontend and backend communicate through a RESTful API — scalable, testable, and deployment-ready.

System Layers
React.js Frontend Port 3000
↕ HTTP / REST
FastAPI Backend Port 8000
↕ SQLAlchemy ORM
PostgreSQL Database Port 5432
↕ Python bindings
OpenCV + Face Recognition AI Engine
  • FastAPI auto-generates interactive API docs at /docs — clients can explore every endpoint live
  • SQLAlchemy ORM handles all DB operations with full migration support via Alembic
  • JWT-based authentication secures every API route — role-based for admin, manager, and employee
  • Face encoding stored in PostgreSQL — recognition happens server-side via Python workers
  • React frontend connects via Axios — environment-variable based API URL for easy deployment switch
  • Backup script included (backup.sh) for automated PostgreSQL dumps

Built-in Security

Security-first by design

Built by a Certified Ethical Hacker — security isn't an afterthought, it's in the architecture from day one.

🔐
JWT Authentication
Stateless token-based auth on every API route. Tokens expire and refresh automatically.
🛡️
Role-Based Access
Admin, Manager, and Employee roles with granular permission control on every endpoint.
🔒
Password Hashing
bcrypt hashing for all stored credentials — never plain-text, never reversible.
🧬
Biometric Data Protection
Face encodings stored as numerical vectors — not images. Can't be reverse-engineered to a photo.
📡
CORS Policy
Strict cross-origin resource sharing rules — only whitelisted origins can call the API.
🗄️
SQL Injection Prevention
All queries go through SQLAlchemy ORM — parameterized, never raw SQL string concatenation.
